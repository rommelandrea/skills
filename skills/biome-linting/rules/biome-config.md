Biome uses `biome.json` (or `biome.jsonc` for comments) as its single configuration file.

## Minimal config

Generated by `npx @biomejs/biome init`:

```json
{
  "$schema": "https://biomejs.dev/schemas/2.0.0/schema.json",
  "linter": {
    "enabled": true,
    "rules": {
      "recommended": true
    }
  },
  "formatter": {
    "enabled": true
  }
}
```

## Formatter options

```json
{
  "formatter": {
    "enabled": true,
    "indentStyle": "space",
    "indentWidth": 2,
    "lineWidth": 80,
    "lineEnding": "lf"
  }
}
```

## Linter rules

Rules are grouped by category: `correctness`, `style`, `suspicious`, `complexity`, `nursery`, `security`, `performance`, `a11y`.

Enable recommended rules for all groups:

```json
{
  "linter": {
    "rules": {
      "recommended": true
    }
  }
}
```

Set severity per group:

```json
{
  "linter": {
    "rules": {
      "correctness": "error",
      "style": "warn",
      "suspicious": "error"
    }
  }
}
```

Override individual rules:

```json
{
  "linter": {
    "rules": {
      "style": {
        "noNonNullAssertion": "off",
        "useConst": "error"
      }
    }
  }
}
```

## Language-specific settings

```json
{
  "javascript": {
    "formatter": {
      "quoteStyle": "single",
      "trailingCommas": "all",
      "semicolons": "always"
    }
  }
}
```

## File control

```json
{
  "files": {
    "ignore": ["dist/**", "node_modules/**", "coverage/**"],
    "maxSize": 1048576
  }
}
```

## VCS integration

Biome can use `.gitignore` patterns automatically:

```json
{
  "vcs": {
    "enabled": true,
    "clientKind": "git",
    "useIgnoreFile": true
  }
}
```

## Overrides

Apply different settings to specific file patterns:

```json
{
  "overrides": [
    {
      "include": ["*.test.ts", "*.spec.ts"],
      "linter": {
        "rules": {
          "suspicious": {
            "noExplicitAny": "off"
          }
        }
      }
    }
  ]
}
```

## Good practices

- Use a single root `biome.json` unless package-level variance is required
- Enable VCS integration to respect `.gitignore` automatically
- Keep overrides minimal — prefer consistent rules across the codebase
- Use `biome.jsonc` if you want inline comments explaining rule choices
